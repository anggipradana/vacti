# Diátaxis Documentation Framework

`docs/` is organized into four modes, each serving a distinct user need:

| Mode        | Dir                 | Purpose                         | Example                      |
| ----------- | ------------------- | ------------------------------- | ---------------------------- |
| Tutorial    | `docs/tutorials/`   | Learning-oriented, step-by-step | "Run your first scan"        |
| How-to      | `docs/how-to/`      | Task-oriented recipes           | "Deploy with Docker Compose" |
| Reference   | `docs/reference/`   | Information-oriented, precise   | "REST API / OpenAPI"         |
| Explanation | `docs/explanation/` | Understanding-oriented          | "Architecture & risk score"  |

Do not mix modes in one document. A tutorial teaches; a how-to assumes competence; reference
describes; explanation discusses trade-offs.
