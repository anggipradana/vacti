# Task: Auth, sessions & API tokens

## Description

Implementasi autentikasi di `@vacti/auth`: password hashing **argon2id**, session berbasis cookie
(httpOnly, SameSite=Lax, rotasi), dan **API token** (hashed + scoped) untuk otomasi/CI eksternal.
Sediakan halaman login/logout + (opsional) first-run create-admin di app `web`.

Menutup baris Feature Parity Checklist: **6.3 (session + API token)**.

## Acceptance Criteria

- [ ] `hashPassword`/`verifyPassword` (argon2id) + `createSession`/`validateSession`/`invalidateSession`.
- [ ] Cookie session httpOnly + SameSite=Lax + Secure (prod); session disimpan di tabel `sessions`.
- [ ] `createApiToken` mengembalikan token plaintext **sekali**, menyimpan hanya hash + scope + label
  + expiry; `validateApiToken` untuk Bearer auth.
- [ ] Halaman `web`: login, logout; first-run create-admin bila belum ada user.
- [ ] tRPC context mengisi `user` dari session; Hono middleware mengisi principal dari Bearer token.
- [ ] Unit test (hash/verify, token lifecycle) + integration test (login → session → akses terproteksi).

## Technical Details

- Library tipis (lucia-style/custom) — bukan framework auth berat.
- Token format: prefix `vct_` + random 32 byte base64url; simpan SHA-256 hash.
- Brute-force guard ringan (rate-limit login) opsional.
- Tidak menangani permission di sini — itu task 005 (RBAC).
