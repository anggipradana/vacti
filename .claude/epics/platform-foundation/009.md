# Task: E2E smoke & coverage gate

## Description

Ikat fondasi dengan **Playwright e2e smoke** yang membuktikan platform hidup: login → buat Project →
buat & cabut API token. Wiring 3-tier testing (unit/integration/e2e) ke gerbang CI sehingga PR
hijau hanya bila semuanya lulus, plus ambang coverage minimal.

Menutup baris Feature Parity Checklist: **8.7 (gate e2e), penutup Epic-1**.

## Acceptance Criteria

- [ ] Playwright e2e: skenario login → create project → create API token → revoke token → logout.
- [ ] E2E memakai service Postgres + app+worker (compose atau test harness) di CI.
- [ ] CI job `e2e` & `integration` (cache:false) tergabung di quality-gate dan **wajib lulus**.
- [ ] Ambang coverage unit minimal (mis. baris kritikal auth/RBAC/queue) ditegakkan; gagal → merah.
- [ ] Dokumen `docs/how-to/run-tests.md` (quick/integration/e2e) + `npm run e2e`.
- [ ] Negative test: Auditor tidak dapat tombol/endpoint `initiate_scans` (UI tersembunyi + 403 server).

## Technical Details

- Reuse Playwright dari task 001; trace on-failure; retas data via seed test.
- Pisahkan `test:quick` (unit, cache) dari `test:e2e`/`test:integration` (no-cache) sesuai nx.json.
- Smoke harus deterministik & < ~2 menit.
