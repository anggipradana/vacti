# Task: pg-boss queue & worker bootstrap

## Description

Sediakan antrian job berbasis **pg-boss** (di Postgres, TANPA Redis) lewat `@vacti/queue` dan
bootstrap proses `worker`: jalankan migrasi DB lalu mulai pg-boss, daftarkan handler. Handler scan
nyata diisi epic **recon-engine**; di sini cukup infrastruktur antrian ber-tipe + 1 job contoh
(health/echo) untuk membuktikan end-to-end.

Menutup baris Feature Parity Checklist: **8.2 (queue di Postgres), 8.1 (worker)**.

## Acceptance Criteria

- [ ] `@vacti/queue`: wrapper `start()`, `enqueue(name, payload)` & `work(name, handler)` ber-tipe
      Zod (payload tervalidasi), schema pg-boss = `pgboss`.
- [ ] Worker entrypoint: validasi env → `drizzle migrate` → `pgboss.start()` → register handler →
      graceful shutdown (SIGTERM/SIGINT) yang menutup koneksi & job.
- [ ] Dukungan retry + visibility timeout (job bertahan saat worker restart) terkonfigurasi.
- [ ] Job contoh `echo` ter-enqueue dari `web`/test dan diproses worker (bukti round-trip).
- [ ] Hook penempatan untuk progress (callback/event) + cancellation (disempurnakan di recon-engine).
- [ ] Integration test (service Postgres): enqueue → worker memproses → state job = completed.

## Technical Details

- pg-boss tabel di schema `pgboss` terpisah dari `public`.
- Konkurensi job dibatasi per-handler (teamSize/teamConcurrency) — siap untuk scan long-running.
- Cancellation pola: simpan jobId di DB; recon-engine memakai `AbortController` + kill child-process.
