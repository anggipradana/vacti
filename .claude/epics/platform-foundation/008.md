# Task: Dockerfile, compose & runtime image

## Description

Kemas runtime minimal: **Dockerfile multi-stage** (base → builder `nx build web+worker` → runtime)
dan **docker-compose** 3-service (`app`, `worker`, `db`). Image worker membawa 4 binary Go
(subfinder/httpx/naabu/nuclei) + nuclei-templates (termasuk WordPress/wordfence) + Chromium
(Playwright) — disiapkan di sini, dipakai recon-engine & reports. Sediakan Makefile/scripts operasional.

Menutup baris Feature Parity Checklist: **8.1, 8.6, dan penyediaan tool untuk 1.1/1.3/1.4/1.6/1.7 + 3.5**.

## Acceptance Criteria

- [ ] `Dockerfile` multi-stage; image `app` ramping (Next.js standalone), image `worker` membawa
      binari recon + nuclei-templates + Chromium.
- [ ] Versi tool ProjectDiscovery di-pin; verifikasi `--version` saat build (fail bila hilang).
- [ ] `docker-compose.yml`: `app`, `worker`, `db (postgres:16)` + healthcheck + depends_on; volume
      data Postgres; env dari `.env`.
- [ ] `docker compose up` → migrasi jalan otomatis (worker) → health endpoint `app` 200.
- [ ] Target ukuran: image app < 1.5 GB termasuk tool + Chromium (tercatat di docs).
- [ ] `Makefile`/scripts: `dev`, `build`, `migrate`, `seed`, `up`, `down`, `test`, `e2e`.
- [ ] `.dockerignore` + dokumentasi deploy ringkas di `docs/how-to/deploy.md`.

## Technical Details

- Base `node:20-slim` (atau `-bookworm` utk Chromium deps). Chromium via `playwright install --with-deps chromium`.
- Binari Go diunduh dari rilis resmi ProjectDiscovery (checksum diverifikasi) ke `/usr/local/bin`.
- Tidak ada Redis/Celery/Ollama/Nginx. Reverse proxy/TLS diserahkan ke deployment (opsional, di docs).
