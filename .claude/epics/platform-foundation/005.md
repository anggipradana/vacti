# Task: RBAC & permission middleware

## Description

Tegakkan **RBAC** di sisi server (bukan hanya UI), mengacu `roles.py` reNgine. Tiga peran —
**SysAdmin / PenetrationTester / Auditor** — dengan matriks permission, plus `requireProjectAccess`
untuk scoping multi-project. Guard dipasang di middleware tRPC dan Hono sehingga semua epic lain
tinggal memakai `requirePermission(...)`.

Menutup baris Feature Parity Checklist: **6.4, 8.4**.

## Acceptance Criteria

- [ ] Definisi permission: `modify_system_config`, `modify_scan_config`, `modify_scan_results`,
  `modify_report`, `initiate_scans`, `modify_targets` (+ baca implisit).
- [ ] Matriks peran: SysAdmin = semua; PenetrationTester = semua kecuali `modify_system_config`;
  Auditor = baca + `modify_report` saja (tolak `initiate_scans`/`modify_*` lain).
- [ ] `requirePermission(perm)` + `requireRole(role)` + `requireProjectAccess(projectId)` sebagai
  middleware tRPC & Hono; gagal → 403, ter-audit ke `audit_log`.
- [ ] Penegakan di server-side terbukti lewat test (Auditor ditolak `initiate_scans`).
- [ ] Helper UI `can(perm)` untuk menyembunyikan kontrol (bukan satu-satunya penjaga).
- [ ] Unit test matriks lengkap + integration test (akses lintas-peran & lintas-project).

## Technical Details

- Peran disimpan per `project_members` (peran per-project) + flag SysAdmin global.
- Matriks permission statis di `@vacti/auth` (single source), diekspor untuk UI & server.
- Semua mutasi sensitif menulis `audit_log`.
