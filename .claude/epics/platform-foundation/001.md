# Task: Monorepo scaffold & toolchain

## Description

Berdirikan workspace **Nx** TypeScript sebagai fondasi monorepo vacti: apps `web` (Next.js 15,
App Router, React 19) dan `worker` (Node), serta libs bersama kosong yang akan diisi task berikutnya:
`@vacti/db`, `@vacti/auth`, `@vacti/core`, `@vacti/config`, `@vacti/queue`, `@vacti/ui`. Termasuk
toolchain TS/ESLint/Prettier(120)/Vitest/Playwright dan npm scripts dasar (`build`, `lint`,
`typecheck`, `test:quick`, `test:integration`, `test:e2e`, `dev`).

Menutup baris Feature Parity Checklist: **8.1 (kerangka), 8.3 (paths)**.

## Acceptance Criteria

- [ ] `nx.json`, `tsconfig.base.json`, `package.json` (workspaces apps/* libs/*) ada & valid.
- [ ] App `web` (Next.js 15 + React 19 + Tailwind) dan `worker` (Node entry) men-`build` lewat Nx.
- [ ] 6 lib stub (`@vacti/{db,auth,core,config,queue,ui}`) dengan `src/index.ts` + path alias di
  `tsconfig.base.json` (`@vacti/*`).
- [ ] Target Nx `typecheck`, `lint`, `test:quick`, `test:integration` (cache:false), `test:e2e`
  (cache:false) terdefinisi dengan `targetDefaults` benar.
- [ ] `nx run-many -t typecheck lint build` hijau pada workspace kosong.
- [ ] ESLint + Prettier (printWidth 120) + Vitest + Playwright terkonfigurasi & jalan.

## Technical Details

- Nx 22+, package manager npm (workspaces). `defaultBase: origin/main`.
- Setiap project punya tag `lang:ts` + tag domain (`scope:web`/`scope:worker`/`scope:lib`).
- Next.js pakai App Router; `next.config` minimal; Tailwind + token desain placeholder.
- Vitest projects per-lib; Playwright config di `apps/web-e2e` (atau `apps/web` test dir).
- Tidak ada logika domain di task ini — murni kerangka yang ber-CI hijau.
