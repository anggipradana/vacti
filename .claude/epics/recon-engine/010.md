# Task: Configurable scan tool options (profile config)

## Description

Buat scan VA bisa dikonfigurasi mendalam dari GUI lewat **Scan Profile**: opsi per-tool (nuclei
User-Agent/tags/templates/rate, httpx UA/threads, dll) + scoping (skip stage, exclude subdomain/path).
Lihat [09-SCAN-CONFIG.md](../../../docs/planning/09-SCAN-CONFIG.md). Maps Feature-Parity 4.5 + Gap §C.

## Acceptance Criteria

- [ ] `scan_profiles.config` (jsonb) menampung opsi lanjutan (userAgent, headers, rateLimit,
      concurrency, retries, nucleiTags/Templates/ExcludeTags, excludeSubdomains, extraArgs per-tool).
- [ ] Adapter args (nuclei/httpx/subfinder/naabu) menghormati opsi; `extraArgs` **allow-listed**
      per-tool (flag tak dikenal dibuang + dilog); tag berbahaya (dos/intrusive/fuzzing) off by default.
- [ ] Pipeline membuang `config.excludeSubdomains` dari daftar host sebelum httpx; skip stage via tools.
- [ ] Halaman **/settings/profiles** (atau /profiles): list + create/edit profil dengan seluruh opsi;
      guard `modify_scan_config`.
- [ ] Unit test arg-builder + allow-list + pipeline-exclude; e2e buat profil custom + start scan.

## Technical Details

- Satu kolom `config` jsonb → minim migrasi saat opsi nambah.
- subfinder provider API keys = item terpisah (perlu provider-config dari vault).
